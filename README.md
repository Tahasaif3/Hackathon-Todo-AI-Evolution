## Project Philosophy

### Human-AI Collaboration Contract

**Humans (Architects)**:
- Write and refine feature specifications
- Review architectural decisions
- Validate tests and approve changes
- Make final decisions on tradeoffs

**AI (Claude Code - Developer)**:
- Generate architecture plans from specs
- Write all implementation code
- Create comprehensive test suites
- Perform refactoring and bug fixes

### Core Principles

1. **Spec First, Code Second**: No feature is implemented without a complete, approved specification
2. **Mandatory Traceability**: Every feature maintains a complete audit trail from ADR to implementation
3. **Test-First Mandate**: Testing is not optional - tests must be generated before or immediately after implementation
4. **Evolutionary Consistency**: Later phases extend but never break earlier phases
5. **Code Generation Only**: All code is generated by AI; manual code writing is prohibited

## Code Generation Policy

âš ï¸ **IMPORTANT**: All code for this project must be generated using Claude CLI. Manual code changes are **strictly prohibited** per the project constitution.

### Why Code Generation?

- **Consistency**: Maintains uniform code style and patterns
- **Quality**: Ensures all quality standards are met
- **Traceability**: Links implementation to specifications
- **Documentation**: Automatic generation of docstrings and comments
- **Testing**: Comprehensive test generation alongside implementation

### Code Generation Process

1. Human writes specification in `/specs/<feature>/spec.md`
2. Human approves specification
3. AI generates plan in `/specs/<feature>/plan.md`
4. AI breaks plan into tasks in `/specs/<feature>/tasks.md`
5. AI generates code from tasks
6. AI writes tests
7. Human reviews and approves
